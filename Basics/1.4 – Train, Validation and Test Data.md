# 1.4 Training, Validation and Test Data

When building a machine learning model, it is important to evaluate how well the model performs on **new unseen data**.
To achieve this, the dataset is usually divided into **three parts**:

```
Training Data
Validation Data
Test Data
```

Each part plays a different role in building and evaluating the machine learning model.

---

# Why Do We Split Data?

If we train and test a model using the **same dataset**, the model may simply **memorize the data** instead of learning real patterns.

This problem is called **Overfitting**.

Example:

If a student memorizes answers from past exam papers but cannot solve new problems, the learning is not real.

Similarly, ML models must be tested on **unseen data** to measure real performance.

---

# Dataset Splitting Overview

The dataset is usually split like this:

```
Dataset
   │
   ├── Training Data
   ├── Validation Data
   └── Test Data
```

Typical percentage splits are:

| Dataset Type | Percentage |
| ------------ | ---------- |
| Training     | 70–80%     |
| Validation   | 10–15%     |
| Test         | 10–15%     |

Example:

If the dataset has **1000 records**:

| Dataset    | Size |
| ---------- | ---- |
| Training   | 700  |
| Validation | 150  |
| Test       | 150  |

---

# 1. Training Data

## Definition

Training data is the portion of the dataset used to **train the machine learning model**.

The model learns patterns and relationships from this data.

Example dataset:

| Study Hours | Exam Score |
| ----------- | ---------- |
| 2           | 40         |
| 4           | 55         |
| 6           | 70         |
| 8           | 90         |

During training, the algorithm learns the relationship:

```
Score = f(Study Hours)
```

---

## Purpose of Training Data

Training data is used to:

* learn patterns
* estimate model parameters
* build predictive models

Most of the dataset is used for training.

Typical proportion:

```
70% – 80%
```

---

# 2. Validation Data

## Definition

Validation data is used to **tune the model and choose the best hyperparameters**.

The model does not directly learn from validation data but uses it to improve configuration.

---

## What Are Hyperparameters?

Hyperparameters are settings chosen **before training the model**.

Examples:

```
Learning rate
Number of trees in Random Forest
Depth of Decision Tree
Number of neural network layers
```

Validation data helps select the **best combination of parameters**.

---

## Example

Suppose we train two models:

| Model   | Accuracy |
| ------- | -------- |
| Model A | 85%      |
| Model B | 90%      |

Using validation data, we choose **Model B**.

---

# 3. Test Data

## Definition

Test data is used to **evaluate the final performance of the model**.

This dataset is **never used during training or validation**.

It simulates how the model will perform in the real world.

---

## Purpose of Test Data

Test data helps measure:

```
Model accuracy
Model reliability
Generalization ability
```

Example:

If a model predicts house prices, the test dataset contains **new houses the model has never seen before**.

---

# Example Workflow

Suppose we have a dataset of **10,000 customer records**.

Split:

| Dataset    | Records |
| ---------- | ------- |
| Training   | 7000    |
| Validation | 1500    |
| Test       | 1500    |

Workflow:

```
1. Train model using training data
2. Tune parameters using validation data
3. Evaluate final model using test data
```

# Overfitting and Underfitting

Splitting data also helps detect two common problems.

---

## Overfitting

Overfitting occurs when the model learns the training data **too well**, including noise.

Characteristics:

```
High training accuracy
Low test accuracy
```

Example:

A model memorizes training data but fails on new data.

Solution:

* More data
* Regularization
* Cross-validation

---

## Underfitting

Underfitting occurs when the model is **too simple to learn patterns**.

Characteristics:

```
Low training accuracy
Low test accuracy
```

Example:

Trying to fit complex data using a simple linear model.

Solution:

* Use more complex models
* Add more features
* Train longer

---

# Cross Validation

Sometimes datasets are small. Instead of a fixed validation split, **cross-validation** is used.

The dataset is divided into multiple parts called **folds**.

Example: **5-Fold Cross Validation**

```
Fold 1 → Test
Fold 2–5 → Train
```

Then the folds rotate:

```
Fold 2 → Test
Fold 1,3,4,5 → Train
```

This process repeats until all folds are used.

Benefits:

```
More reliable evaluation
Better use of limited data
```

---

# Summary

Dataset splitting is a critical step in machine learning.

The dataset is divided into three parts:

| Dataset         | Purpose              |
| --------------- | -------------------- |
| Training Data   | Train the model      |
| Validation Data | Tune hyperparameters |
| Test Data       | Evaluate final model |

Typical split:

```
Training = 70–80%
Validation = 10–15%
Test = 10–15%
```

Proper dataset splitting ensures the model:

```
Learns patterns
Avoids overfitting
Performs well on new data
```
