# 1.5 Underfitting and Overfitting in Machine Learning

Machine learning models should learn **useful patterns** from training data.

If the model learns **too little** or **too much**, problems occur.

* **Underfitting** → Model is too simple and fails to capture patterns.
* **Overfitting** → Model learns patterns **plus noise** from training data.

A **good machine learning model** finds the correct balance:

* Complex enough to capture real patterns
* Simple enough to avoid memorizing noise

This balance is one of the **most important goals in machine learning**.

---

# What is Underfitting?

**Underfitting** occurs when a model fails to learn the important patterns from the data.

### Characteristics

* Performs **poorly on training data**
* Performs **poorly on testing data**
* Model is **too simple**

Example:

Trying to fit a **straight line** to data that actually follows a **curve**.

The model simply **cannot represent the real relationship**.

---

# Causes of Underfitting

Underfitting can happen due to several reasons:

* Model is too simple
* Very high regularization
* Important features are missing
* Not enough training time
* High bias

---

# Example of Bias

Bias means the model makes **strong assumptions** about the data.

Example assumption:

> “All birds are small and can fly.”

Because of this assumption, the model cannot recognize birds like:

* Ostrich
* Penguin

These birds are **large and cannot fly**, so the model gives incorrect predictions.

This strong assumption leads to **biased predictions**.

---

# Bias–Variance Inside Underfitting

Underfitting mainly occurs due to **High Bias**.

### Properties

High bias means:

* Model makes strong assumptions
* Ignores complex patterns
* Learns overly simple relationships

Variance becomes **low** because the model barely changes even if the data changes.

Conceptually:

Underfitting = **High Bias + Low Variance**

---

# What is Overfitting?

**Overfitting** occurs when a model learns **too much from the training data**, including:

* Noise
* Random fluctuations
* Outliers

Instead of learning the **true pattern**, the model **memorizes the training data**.

### Characteristics

* Performs **extremely well on training data**
* Performs **poorly on new (test) data**

---

# Example of Variance

Variance refers to how **sensitive a model is to training data changes**.

High variance means:

* Model reacts strongly to small changes in the dataset
* Model learns noise as if it were a real pattern

---

# Bias–Variance Inside Overfitting

Overfitting mainly occurs due to **High Variance**.

Properties:

* Model is extremely flexible
* Learns noise
* Low bias
* Very high variance

Conceptually:

Overfitting = **Low Bias + High Variance**

---

# Visual Understanding of Underfitting and Overfitting

Imagine we want to predict **house price based on house size**.

The real relationship is usually **curved**, not perfectly straight.

Three types of models can occur.

---

## Underfitting

The model is **too simple**.

Example:

A straight line trying to fit curved data.

Result:

* Important patterns are missed
* Poor predictions

---

## Proper Fit (Ideal Model)

The model is **moderately complex**.

Result:

* Captures the real trend
* Ignores random noise
* Works well on new data

---

## Overfitting

The model becomes **too complex**.

Result:

* Fits every training point
* Learns noise
* Performs poorly on new data

---

# Python Tutorial: Visualizing Underfitting, Proper Fit, and Overfitting

Now we will **visualize these concepts using Python**.

---

# Step 1 — Import Libraries

```python
import numpy as np
import matplotlib.pyplot as plt
```

NumPy helps generate and manipulate numerical data.

Matplotlib helps visualize the results.

---

# Step 2 — Create a Sample Dataset

```python
np.random.seed(42)

x = np.linspace(0, 10, 30)
y = np.sin(x) + np.random.normal(0, 0.2, 30)
```

Explanation:

`np.linspace(0,10,30)`
Creates **30 evenly spaced values** between 0 and 10.

`np.sin(x)`
Creates a **smooth curved relationship**.

`np.random.normal(0,0.2,30)`
Adds **random noise** to simulate real-world data.

---

# Step 3 — Create Smooth Values for Plotting

```python
x_curve = np.linspace(0, 10, 100)
```

We create **100 points** to make the predicted curve smooth.

---

# Step 4 — Train Three Different Models

### Underfitting Model (Degree 1)

```python
coef_under = np.polyfit(x, y, 1)
poly_under = np.poly1d(coef_under)
y_under = poly_under(x_curve)
```

Degree 1 means **straight line**.

---

### Proper Fit Model (Degree 4)

```python
coef_good = np.polyfit(x, y, 4)
poly_good = np.poly1d(coef_good)
y_good = poly_good(x_curve)
```

Degree 4 captures the **true curve** without being too complex.

---

### Overfitting Model (Degree 15)

```python
coef_over = np.polyfit(x, y, 15)
poly_over = np.poly1d(coef_over)
y_over = poly_over(x_curve)
```

Degree 15 is **too flexible** and starts fitting noise.

---

# Step 5 — Plot All Three Models

```python
plt.figure(figsize=(15,5))

plt.subplot(1,3,1)
plt.scatter(x,y)
plt.plot(x_curve,y_under)
plt.title("Underfitting")

plt.subplot(1,3,2)
plt.scatter(x,y)
plt.plot(x_curve,y_good)
plt.title("Proper Fit")

plt.subplot(1,3,3)
plt.scatter(x,y)
plt.plot(x_curve,y_over)
plt.title("Overfitting")

plt.tight_layout()
plt.show()
```

Output:

You will see **three graphs side by side**:

1️⃣ Straight line → Underfitting
2️⃣ Smooth curve → Ideal model
3️⃣ Wiggly curve → Overfitting

# Second Important Visualization in Machine Learning

One of the **most important graphs in ML research** shows:

**Training Error vs Test Error vs Model Complexity**

This graph explains **why overfitting happens mathematically**.

---

# Concept

As model complexity increases:

* **Training error keeps decreasing**
* **Test error first decreases then increases**

Why?

Because complex models eventually start **learning noise**.

---

# Python Code to Visualize Bias–Variance Tradeoff

```python
complexity = np.arange(1,20)

training_error = 1/(complexity) + 0.05
test_error = 0.3 + ((complexity-7)**2)/100

plt.plot(complexity, training_error,label="Training Error")
plt.plot(complexity, test_error,label="Test Error")

plt.xlabel("Model Complexity")
plt.ylabel("Error")
plt.title("Bias-Variance Tradeoff")

plt.legend()
plt.show()
```

---

# What This Graph Shows

Left side:

Model is too simple → **Underfitting**

Middle:

Best balance → **Optimal model**

Right side:

Model too complex → **Overfitting**

---

# Final Key Idea

The goal of machine learning is:

**Find the model complexity where test error is minimum.**

That is the **optimal balance between bias and variance**.
